## The 10 Fundamental Rules

These aren't suggestions. They're what separates a frustrating experience from a life-changing one.

| # | Rule | Why It Matters |
|---|------|---------------|
| 1 | **One source of truth** | Every piece of data lives in ONE place. Not scattered across 5 files. Your database, your CLAUDE.md, your state file — each owns its domain. |
| 2 | **Never lose anything** | Context-survival + session-end-enforcer + state file = impossible to lose work. Sessions auto-save at 60%, 70%, 80%. After compaction, auto-resume. |
| 3 | **Visual-first output** | Max 3 sentences before a table, box frame, or checklist. If you can't scan it in 5 seconds, it's wrong. |
| 4 | **Gate everything** | Pre-flight: are inputs correct? Post-flight: is output good? If Claude can skip a check, it will. Gates prevent that. |
| 5 | **Docs before guessing** | Always read official documentation before advising on any tool. Never guess at APIs. |
| 6 | **Parallel by default** | Free parallel tool calls > expensive agent spawns. Batch reads, searches, and checks. Save agents for complex work. |
| 7 | **Everything repeatable = a skill** | Did something twice? It's a skill. Package it so you (or Claude) can run it perfectly every time. |
| 8 | **Tag and categorize everything** | Frameworks, keywords, layers — structured data is searchable data. Unstructured data is lost data. |
| 9 | **Fix it, don't explain it** | Try 3 approaches before asking. Don't explain blockers — solve them. |
| 10 | **Ship, don't tinker** | Track your dangerous-to-system ratio. If you've been building internal stuff for 60+ min without any visible output — something's wrong. |

## What's In The Kit

### Hooks (Auto-Run, Zero Effort)

Hooks are scripts that fire automatically during your sessions. Install once, never think about them again.

| Hook | Pain It Kills | How It Works |
|------|--------------|--------------|
| `context-survival.js` | "My session died mid-task" | Auto-saves work state at 60/70/80% context usage. After auto-compaction, reads it back. **Your session literally never dies.** |
| `smart-boot.js` | "Where was I?" | At session start: checks for recent state file, detects your project from directory, shows git branch. Picks up exactly where you left off. |
| `cost-guardian.js` | "Burned my whole weekly limit" | Warns at 50%, 70%, 85% of weekly usage. Tracks expensive agent spawns. You'll never be surprised again. |
| `ship-tracker.js` | "Spent all day, shipped nothing" | Classifies work as "dangerous" (pushed code, sent emails, created content) vs "system" (configs, refactors). Alerts after 60 min of system-only work. |
| `session-end-enforcer.js` | "Forgot to save my work" | Commands (not suggests) backup sequence when session is ending. Shows verification table of what's saved and what's not. |

- What you're struggling with
- What your day looks like

## Clone Yourself

This is where it gets really powerful.

**Everyone should create their own AI clone.** A clone is a document that teaches Claude exactly who you are — how you think, how you talk, what you know, what you care about, your frameworks, your opinions, your style.

### Why Clone Yourself?

- Claude stops being generic and starts being **you**
- It remembers your preferences, vocabulary, and decision-making patterns
- It can draft emails, messages, and content **in your voice**
- It becomes a thinking partner that knows your context
- New team members can "talk to your clone" to learn how you operate

### How to Build Your Clone (Simple Version)

Create this file: `~/.claude/skills/my-clone/SKILL.md`

```markdown
# [Your Name] — AI Clone

## Who I Am
[Your role, what you do, what you're building]

## How I Think
[Your decision-making process, what you prioritize]

## How I Communicate
[Short sentences? Formal? Casual? Direct? Do you use analogies?]

## What I Know
[Your expertise areas, frameworks you use, tools you know]

## My Opinions
[Strong opinions on your field, things you believe in]

## My Style
[How you write emails, how you give feedback, your catchphrases]

## Never Do
[Things that don't sound like you, approaches you'd never take]

## Always Do
[Your non-negotiables, your standards, your patterns]
```

### How to Build Your Clone (Power Version)

Feed Claude everything:
1. Your emails (it learns your writing voice)
2. Your Slack messages (it learns your casual voice)
3. Your documents/notes (it learns your thinking)
4. Your personality test results (it learns your wiring)
5. Voice recordings/transcripts (it learns how you actually talk)

Then say: **"Build a clone of me based on everything you've seen."**

Claude will synthesize it into a structured clone document. The more you feed it, the more accurate it gets.

**Pro tip:** Your clone is a living document. Update it as you evolve. The best clones have hundreds of data points — quotes, decisions, reactions, preferences — organized into searchable sections.

## Building Skills (Make Everything Repeatable)

Here's a rule that will change how you use Claude Code forever:

**If you did something twice, it's a skill. Package it.**

A skill is a reusable instruction file that Claude reads when you invoke it. Instead of explaining what you want every time, you say `/my-skill` and it runs perfectly.

### How to Create a Skill

Official docs: https://docs.anthropic.com/en/docs/claude-code/skills

```
~/.claude/skills/my-skill/
├── SKILL.md          <- Instructions Claude follows
└── references/       <- Supporting docs, templates, examples
    └── template.md
```

Basic SKILL.md format:
```markdown
---
name: my-skill
description: "What this skill does in one sentence."
invocation: "/my-skill"
---

# My Skill

## What This Does
[Explain the task]

## Steps
1. [Step 1]
2. [Step 2]
3. [Step 3]

## Output Format
[What the result should look like]
```

**The mindset:** Every workflow you repeat is a skill waiting to be built. The more skills you create, the more powerful Claude Code becomes — permanently.

## MCP Servers — Always-On Superpowers

MCP (Model Context Protocol) servers give Claude direct access to your tools. Once installed, they're always on — Claude can use them anytime without you doing anything.

**Think of MCPs like giving Claude keys to your apps.** Gmail MCP = Claude can read your email. Notion MCP = Claude can write to your database. Calendar MCP = Claude knows your schedule.

| **Google Calendar** | View, create, manage events | `claude mcp add gcal -- npx @anthropic/claude-google-calendar-mcp` |

## FAQ

**Q: Will this conflict with my existing setup?**
A: No. The installer skips existing files. If you have hooks already, merge the settings.json manually.

**Q: Mac or Windows — does it matter?**
A: The hooks are Node.js — platform-independent. Install commands differ (brew vs apt), but everything else is identical.

**Q: How do I update the kit?**
A: `cd claude-code-starter-kit && git pull && bash install.sh` — won't overwrite your customizations.

**Q: Can I remove a hook?**
A: Delete from `~/.claude/hooks/` and remove from `~/.claude/settings.json`.

**Q: I use Obsidian, not Notion.**
A: Kit is tool-agnostic. Obsidian works great — just set your vault path in CLAUDE.md and Claude reads/writes directly via filesystem.

**Q: What is a "skill" and do I need one?**
A: A skill = a reusable instruction file. If you do something more than once, make it a skill. See [Building Skills](#building-skills-make-everything-repeatable). Official docs: https://docs.anthropic.com/en/docs/claude-code/skills

**Q: Where should my API keys live?**
A: In `~/.claude/.env` — NEVER in your code, NEVER committed to git, NEVER shared with anyone. Each person creates their own keys.

**Q: I'm a total beginner. Is this too advanced?**
A: No. The Getting to Know You form lets you set your depth level. If you picked "Just the basics," Claude will keep things simple. The kit works for beginners and power users — it adapts to you.

**Q: What if I break something?**
A: That's what gates are for. Pre-flight checks catch bad inputs. Post-flight checks catch bad outputs. And the context-survival hook means even if your session crashes, you pick up right where you left off. You literally cannot lose work.

**Q: How do I build my own clone?**
A: See [Clone Yourself](#clone-yourself). Start simple — name, role, how you think, how you communicate. Feed it more over time. Your clone gets better every week.

**Q: How much will this cost me per month?**
A: $20/mo minimum (Claude Code subscription). $40/mo recommended (add Cursor). See the [Budget Estimate](#budget-estimate) for full breakdown.
