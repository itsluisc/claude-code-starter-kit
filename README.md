### What Are The Files Claude Creates?

If you're not a developer, you might see unfamiliar file types. Don't panic:

| File Type | What It Is | Real-World Analogy |
|-----------|-----------|-------------------|
| `.md` (Markdown) | A text file with simple formatting | Like a Word doc, but simpler. Headlines, bold, lists. That's it. |
| `.json` | A structured data file | Like a spreadsheet — organized information Claude can read and write |
| `.js` (JavaScript) | A script that runs automatically | Like a recipe — a list of steps the computer follows |
| `.py` (Python) | Another type of script | Same as JavaScript, just a different language |
| `.env` | A file that stores your private API keys | Like a vault combination — never share it, never post it online |

**You don't need to understand these to use Claude Code.** But knowing what they are means you'll never be scared when you see them.

## The 10 Fundamental Rules

These aren't suggestions. They're baked into every piece of this kit. They're the difference between a frustrating AI experience and a transformational one.

| # | Rule | Plain English |
|---|------|--------------|
| 1 | **One source of truth** | Every piece of data lives in ONE place. Not scattered. Not duplicated. One home. |
| 2 | **Never lose anything** | Auto-save at 60%, 70%, 80%. Auto-resume after crashes. Your work is always safe. |
| 3 | **Visual-first** | Max 3 sentences before a table, diagram, or checklist. If you can't scan it in 5 seconds, it's wrong. |
| 4 | **Gate everything** | Check before doing. Check after doing. Never show broken output. Never skip verification. |
| 5 | **Docs before guessing** | Always read official documentation first. Never guess at how something works. |
| 6 | **Parallel by default** | Do multiple things at once when possible. Don't wait when you don't have to. |
| 7 | **Everything repeatable = a skill** | Did something twice? Package it. One command, perfect every time, forever. |
| 8 | **Tag and categorize** | Label everything. Structured data is findable data. Unstructured data is lost data. |
| 9 | **Fix it, don't explain it** | Try 3 approaches before asking for help. Solve problems, don't describe them. |
| 10 | **Ship, don't tinker** | If you've been "setting up" for an hour with nothing to show — ship something visible. |

## What's In The Kit

### Hooks (Auto-Run, Zero Effort)

Hooks are small scripts that run automatically during your sessions. You install them once and never think about them again. They protect you in the background.

| Hook | Pain It Kills | What It Does |
|------|--------------|--------------|
| `context-survival.js` | "My session died mid-task" | Auto-saves your work at 60/70/80% memory usage. If the session resets, it reads the save file and continues. **Your session literally never dies.** |
| `smart-boot.js` | "Where was I?" | When you start Claude Code: checks for a recent save, detects your project, shows your git branch. Picks up where you left off. |
| `cost-guardian.js` | "Burned my whole weekly limit" | Warns you at 50%, 70%, 85% of your weekly usage. You'll never wake up to zero credits again. |
| `ship-tracker.js` | "Spent all day, shipped nothing" | Tracks whether you're creating visible output or just doing internal setup. Alerts after 60 min of setup-only work. |
| `session-end-enforcer.js` | "Forgot to save my work" | When your session is ending, it forces a backup. Shows you exactly what's saved and what's not. |

### CLAUDE.md Template

Your `CLAUDE.md` is the instruction manual Claude reads at the start of every session. Think of it as Claude's cheat sheet about you. The template includes:
- **Visual-first output rules** — tables, box frames, checklists (no walls of text)
- **Context survival protocol** — how to handle save/resume
- **Gate protocol** — check before doing, check after doing
- **Parallel execution** — do multiple things at once
- **Docs-first** — always read documentation before guessing
- **No excuses** — solve problems, don't explain why they exist

### Reference Docs

| Doc | What It Teaches Claude |
|-----|----------------------|
| `gate-protocol.md` | How to verify inputs and outputs on every operation |
| `qc-backup-protocol.md` | Where to save backups and how to verify them |
| `documentation-first.md` | Always fetch official docs before advising on any tool |
| `quality-gates.md` | Pre/mid/post-flight checklists for any operation |
| `tool-stack.md` | Which tools to use for what (transcription, video, search, etc.) |

## Building Your AI Relationship

This is the part nobody teaches. And it's the most important part.

**AI is not a search engine. It's a relationship.** The more it knows about you, the better it gets. The more you trust it, the more you can do together.

### Your First Real Conversation

After installing the kit, just talk to Claude. Seriously. Not a question. A conversation:

- "Here's what I'm building and why..."
- "Here's what frustrates me about my work..."
- "Here's what I wish I could automate..."
- "Here's what scares me about AI..."
- "Here's what my typical day looks like..."

**Don't structure it. Just blab.** Talk like you're talking to a friend who happens to be a genius engineer. Claude will organize everything for you.

### The Comfort Curve

```
Week 1:  "Can you do this?"        → Discovery. Testing the waters.
Week 2:  "Do this."                → Trust. You stop being polite.
Week 3:  "Do this, but my way."    → Personalization. Your style, your rules.
Week 4:  "You already know."       → Partnership. Claude anticipates you.
Month 2: "What should I do next?"  → Delegation. Claude becomes strategic.
Month 3: "Handle it."              → Autonomy. You focus on what only YOU can do.
```

**Every stage is valuable. Don't rush.** But know that it gets dramatically better over time — and the Getting to Know You answers are what accelerate this.

### The Transformation (Information vs. Transformation)

Most people use AI for **information** — "What is X? How do I do Y?"

That's level one. The real power is **transformation**:

| Information (5%) | Transformation (100%) |
|-------------------|----------------------|
| "What's a good subject line?" | "Here are my last 50 emails. Learn my voice. Write all my emails from now on." |
| "How do I edit a video?" | "Here's my style. Here's my footage. Edit it the way I would." |
| "What should I post today?" | "You know my niche, my audience, my voice. Give me 3 content ideas with hooks." |

**The shift:** Stop asking questions. Start giving context. The more Claude knows, the less you need to say, and the more it delivers.

This is where it gets powerful. Everyone should eventually create their own AI clone.

**A clone is a document that teaches Claude who YOU are** — how you think, how you talk, what you know, what you care about, your opinions, your style.

### Why?

- Claude stops being generic and starts being **you**
- It drafts emails, messages, content **in your voice**
- New team members "talk to your clone" to learn how you operate

```markdown
# [Your Name] — AI Clone

[How you write emails, give feedback, make decisions]

[Things that don't sound like you, approaches you'd never take]
```

## Building Skills (Everything Repeatable = A Skill)

Here's the rule that changes everything:

**If you did something twice, it's a skill. Package it. Never do it manually again.**

A skill is a reusable instruction file. Instead of explaining what you want every time, you say `/my-skill` and it runs perfectly. Forever.

### How to Create a Skill

Official docs: https://docs.anthropic.com/en/docs/claude-code/skills

```
~/.claude/skills/my-skill/
├── SKILL.md          <- Instructions Claude follows
└── references/       <- Supporting docs, templates, examples
    └── template.md
```

Basic SKILL.md:
```markdown
---
name: my-skill
description: "What this skill does in one sentence."
invocation: "/my-skill"
---

# My Skill

## What This Does
[Explain the task]

## Steps
1. [Step 1]
2. [Step 2]
3. [Step 3]

## Output Format
[What the result should look like]
```

### What Should Be A Skill?

Go back to question 8 in the Getting to Know You form. Everything you hate doing? **That's a skill.**

| What You Hate Doing | Skill That Fixes It |
|---------------------|-------------------|
| "Writing the same emails over and over" | `/email-draft` — drafts in your voice |
| "Summarizing meetings" | `/meeting-notes` — transcribe + summarize |
| "Coming up with content ideas" | `/content-ideas` — generates ideas for your niche |
| "Writing commit messages" | `/commit` — auto-generates from your changes |
| "Code reviews" | `/review` — runs your checklist every time |
| "Morning planning" | `/morning` — checks calendar, tasks, priorities |
| "Client reports" | `/report` — generates from your data |
| "Invoice creation" | `/invoice` — templates + auto-fill |
| "Social media posting" | `/post` — write, format, schedule |
| "Research on a topic" | `/research` — search, summarize, save to notes |

### Skill Best Practices

Every skill you build should follow these rules:

1. **One skill, one job.** Don't make a mega-skill. Make focused skills.
2. **Clear input/output.** What goes in? What comes out? Define both.
3. **Gate it.** Pre-flight check (are inputs valid?) → Do the work → Post-flight check (is output good?)
4. **Version it.** Put a version number in your SKILL.md. Update when you improve it.
5. **Push to GitHub.** Skills only on your machine = skills that can die. Back them up.

### From "I Hate This" to "It's Handled" — In 5 Minutes

Here's how fast you can eliminate a pain point:

1. **Identify:** "I hate writing LinkedIn posts"
2. **Tell Claude:** "Build me a skill called `/linkedin` that takes a topic and writes a post in my style"
3. **Claude creates the SKILL.md** with your voice, your format, your rules
4. **Test it:** `/linkedin` → "Topic: How AI saved me 20 hours this week"
5. **Done.** You never write a LinkedIn post manually again.

**That's 5 minutes to eliminate a task you've been doing for years.** Multiply that across 10, 20, 50 skills and you start to understand why we say "save 20 hours a week."

```bash
brew install sqlite3           # Local databases
brew install wget              # Download files
brew install tree              # Visual directory structure

# CONTENT CREATORS
brew install yt-dlp            # Download YouTube videos

```

### Windows (WSL2)

```bash
# Install WSL2 first (PowerShell as admin):
# wsl --install

# Then in WSL2 terminal:
sudo apt update && sudo apt install -y \
  nodejs npm git jq ripgrep sqlite3 wget tree ffmpeg imagemagick

# GitHub CLI
curl -fsSL https://cli.github.com/packages/githubcli-archive-keyring.gpg | sudo dd of=/usr/share/keyrings/githubcli-archive-keyring.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/githubcli-archive-keyring.gpg] https://cli.github.com/packages stable main" | sudo tee /etc/apt/sources.list.d/github-cli.list > /dev/null
sudo apt update && sudo apt install gh

# Python tooling
sudo apt install -y python3-pip
pip install uv yt-dlp
```

## FAQ

**Q: Will this break anything?**
A: No. The installer only adds files — never deletes or overwrites. Safe to re-run.

**Q: Mac or Windows — does it matter?**
A: Hooks are platform-independent (Node.js). Install commands differ, everything else is the same.

**Q: How do I update?**
A: `cd claude-code-starter-kit && git pull && bash install.sh`

**Q: I'm a total beginner. Is this too advanced?**
A: No. Find your archetype above (Explorer, Builder, Architect, Transformer) and follow that path. The kit adapts to you.

**Q: What if I break something?**
A: You can't. Git tracks every change. Hooks auto-save your work. Session-end enforcer catches everything. Relax.

**Q: What is a "skill"?**
A: A reusable instruction file. Anything you do more than once = a skill. Official docs: https://docs.anthropic.com/en/docs/claude-code/skills

**Q: Where do my API keys go?**
A: `~/.claude/.env` — never in code, never in git, never shared.

**Q: What are these .md and .json files?**
A: `.md` = text with formatting (like a simple Word doc). `.json` = organized data (like a spreadsheet). See [What Are The Files](#what-are-the-files-claude-creates).

**Q: How much does all this cost?**
A: $20/mo minimum. $40/mo recommended. See [Budget Estimate](#budget-estimate).

**Q: What's Level 2 and 3?**
A: Level 2 = Build your Second Brain (organize your knowledge). Level 3 = Skills School (every problem becomes a solved skill). They build on this foundation.

**Q: Can I use this for my team?**
A: Yes. Everyone installs the kit with their own API keys. The Getting to Know You form personalizes it per person.
