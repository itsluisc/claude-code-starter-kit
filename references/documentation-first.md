# Documentation-First Protocol

**BEFORE advising, configuring, troubleshooting, or using ANY software, hardware, API, tool, library, framework, or service — FIND AND READ THE OFFICIAL DOCUMENTATION FIRST.**

Training data = snapshot. Docs = truth. The snapshot loses.

## The Rule
1. **Identify** the product/tool/service — name + version if possible
2. **Find official docs** — WebSearch or WebFetch the official site's documentation
3. **Read the relevant section** — don't skim, actually read
4. **THEN and only then** — give advice or recommendations

## Applies to EVERYTHING
- Software (CapCut, Premiere, FFmpeg, yt-dlp, Claude, Notion, etc.)
- Hardware (Acasis dock, Plaud, iPhone features, cameras, etc.)
- APIs (ElevenLabs, HeyGen, OpenAI, Anthropic, etc.)
- Libraries (pymupdf, python-docx, Remotion, etc.)
- Services (GitHub, Vercel, Google Drive, Telegram, etc.)
- MCP servers and tools

## Search Priority
1. Official product/company website docs section
2. GitHub README / wiki (for open source)
3. API reference pages
4. Recent changelog / release notes

## Origin
Agent Luis gave a Code Council review of a hardware product without checking the product's actual documentation page first. Docs have the truth.
